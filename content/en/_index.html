{{< /blocks/cover >}} {{% blocks/lead color="primary" %}} Beyond Gentoo. MocaccinoOS uses [Luet](https://luet-lab.github.io/docs) as a package manager, which is completely static and fully based on containers. It comes in two variants: * Mocaccino Micro: Musl-based Linux From Scratch using only [Luet](https://luet-lab.github.io/docs). Suited for cloud and containers, but more to come as a Desktop flavor. * Mocaccino Desktop: Gentoo-based (former Sabayon branch) suited for Desktop environments. {{% /blocks/lead %}} {{< blocks/section color="dark" >}} {{% blocks/feature icon="fa-lightbulb" title="No frills" %}} MocaccinoOS uses a layered approach for system and common packages: smoother upgrades so you can focus on the things that really matter. {{% /blocks/feature %}} {{% blocks/feature icon="fab fa-github" title="Contributions welcome!" url="https://github.com/MocaccinoOS" %}} MocaccinoOS is a work-in-progress. Focus on the things that matter.

MocaccinoOS Desktop uses a layered approach.
Most common system packages and desktop environments (DE) are packaged as single installable layers, along with libraries.
The system set is only composed by 2 core layers

Completely out of sight !

MocaccinoOS Micro is a LFS Musl OS based not on layers, but on a tiny core package set, suitable for cloud and embedded environment.
Easy to build packages on.

Customize your experience

A declarative suite of tools to set your box in the cloud or in a server environment
{{< /blocks/section >}} {{< blocks/section color="primary" >}} {{% blocks/feature icon="fa-truck" title="Luet" %}} [luet](https://github.com/mudler/luet) is a small-footprint static Golang package manager. It uses SAT and RL approaches to solve package dependencies. {{% /blocks/feature %}} {{% blocks/feature icon="fa-wrench" title="Yip" %}} [yip](https://github.com/mudler/yip) is a small-footprint distro-agnostic cloud-init style tool. Statically built which executes and sets Linux system configurations. {{% /blocks/feature %}} {{% blocks/feature icon="fa-lock" title="Declarative user management" %}} With [entities](https://github.com/mudler/entities), users and groups in your systems are formally declared and can be carried over to other systems easily. {{% /blocks/feature %}} {{< /blocks/section >}}
